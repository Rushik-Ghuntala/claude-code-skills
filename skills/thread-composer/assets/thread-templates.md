# Thread Templates

## Listicle Template

```
1/ [NUMBER] [things/tools/tips/lessons] that [value proposition]:

[Optional: brief credibility line]

🧵👇

2/ 1. [Item name]

[Why it matters in 1-2 sentences]

[Specific example or metric]

3/ 2. [Item name]

[Why it matters]

[Example]

...continue for each item...

N/ That's the list.

[1-sentence summary of the value]

[CTA: "Which ones are you using?" or "What would you add?"]

Follow for more [topic] content.
```

## Story Arc Template

```
1/ [Time reference] I [dramatic situation].

[Current contrasting state].

Here's the full story: 🧵

2/ The setup:

[Context — who you were, what you were doing]

[The inciting incident]

3/ The low point:

[What went wrong]

[How it felt — be vulnerable]

4/ The turning point:

[What changed — a realization, advice, event]

[Why this was different from previous attempts]

5/ What I did differently:

[Specific actions taken]

[Timeline]

6/ The result:

[Concrete outcomes — numbers, metrics, status]

[Before vs. after comparison]

7/ Key lesson:

[The main takeaway — make it universal]

[How others can apply it]

N/ If you're in the [low point] phase right now:

[Encouragement + specific advice]

[CTA: Follow for more honest stories like this]
```

## Tutorial Template

```
1/ How to [achieve specific outcome] in [timeframe]:

A step-by-step guide (with code).

🧵👇

2/ What you'll build:

[Description of end result]

Prerequisites:
- [Tool/knowledge 1]
- [Tool/knowledge 2]

3/ Step 1: [Action]

[Code block or command]

[What this does and why]

4/ Step 2: [Action]

[Code block or command]

[Explanation]

...continue for each step...

N-1/ The final result:

[Screenshot or output]

[What you've achieved]

N/ Bookmark this for your next project.

[Summary of what was built]

Follow for more [topic] tutorials.

[Self-reply: links to repo, docs, related content]
```

## Case Study Template

```
1/ We [achieved impressive result].

Here's exactly how (and what went wrong first):

🧵

2/ The problem:

[What we were dealing with]

[Key metric: "Our API took Xs to respond"]

3/ First attempt:

[What we tried first]

[Why it didn't work — specific reason]

4/ The breakthrough:

[What insight or approach changed things]

[Why this was different]

5/ Implementation:

[What we actually built/changed]

[Key technical decisions]

6/ Results:

Before: [metric]
After: [metric]

[Additional benefits we didn't expect]

7/ Lessons:

[2-3 universal takeaways]

[What we'd do differently next time]

N/ Have a similar challenge?

Reply with your setup — I'll share specific advice.

Follow for more engineering case studies.
```

## Comparison Template

```
1/ I used both [X] and [Y] in production.

Here's my honest comparison after [timeframe]:

🧵

2/ Context:

[What I built, scale, team size]

[Why I tried both]

3/ [Tool X] — Strengths:

[2-3 specific strengths with examples]

4/ [Tool X] — Weaknesses:

[2-3 specific weaknesses with examples]

5/ [Tool Y] — Strengths:

[2-3 specific strengths with examples]

6/ [Tool Y] — Weaknesses:

[2-3 specific weaknesses with examples]

7/ Head-to-head:

[Key dimension 1]: [X] wins
[Key dimension 2]: [Y] wins
[Key dimension 3]: Tie

8/ My verdict:

Use [X] when: [specific scenario]
Use [Y] when: [specific scenario]

[Overall recommendation]

N/ Which are you using?

Agree or disagree with my take?

Follow for more honest comparisons.
```
