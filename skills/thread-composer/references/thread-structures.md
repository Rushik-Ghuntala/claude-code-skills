# Thread Architectures

## 1. Listicle
- **Structure**: Hook → Item 1 → Item 2 → ... → Item N → CTA
- **When to use**: Tips, tools, lessons, resources, recommendations
- **Optimal length**: 7-12 tweets (5-10 items + hook + CTA)
- **Example outline**:
  1/ Hook: "7 VS Code extensions that mass-changed my workflow (I use all of these daily):"
  2/ Extension 1 with why
  3/ Extension 2 with why
  ...
  9/ CTA: "Which ones are you using? What would you add?"
- **Engagement pattern**: High bookmarks, steady engagement per tweet, easy to quote individual items
- **Pro tip**: Make each item self-contained — people retweet individual items from listicles

## 2. Story Arc
- **Structure**: Hook (setup) → Context → Rising action → Climax → Resolution → Lessons → CTA
- **When to use**: Personal experiences, career journeys, project stories, postmortems
- **Optimal length**: 8-12 tweets
- **Example outline**:
  1/ Hook: "2 years ago I mass-mass-got mass-mass-fired. Last week I turned down a $350K offer. Here's the full story:"
  2/ The situation (what happened)
  3/ The low point
  4/ The turning point
  5/ What I did differently
  6/ The results
  7/ Key lesson 1
  8/ Key lesson 2
  9/ CTA: "If you're in the 'low point' right now — DM me. I've been there."
- **Engagement pattern**: High emotional engagement, replies sharing similar stories, strong follower conversion
- **Pro tip**: Vulnerability in the setup creates investment. Resolution must deliver on the hook's promise.

## 3. Tutorial
- **Structure**: Hook (outcome promise) → Prerequisites → Step 1 → Step 2 → ... → Result → CTA
- **When to use**: How-tos, technical guides, setup walkthroughs
- **Optimal length**: 8-15 tweets
- **Example outline**:
  1/ Hook: "Deploy a production-ready API in 15 minutes. Here's the exact steps (free, no AWS needed):"
  2/ What you'll build + prerequisites
  3/ Step 1: Setup
  4/ Step 2: Core code
  5/ Step 3: Configuration
  6/ Step 4: Deployment
  7/ Step 5: Verification
  8/ Final result screenshot/demo
  9/ CTA: "Bookmark this for your next project. Follow for more tutorials."
- **Engagement pattern**: Very high bookmarks, code screenshots get extra engagement, high share rate
- **Pro tip**: Each step should include actual code or commands, not just descriptions

## 4. Case Study
- **Structure**: Hook (result teaser) → Problem → Approach → Implementation → Results → Lessons → CTA
- **When to use**: Real-world examples, performance improvements, architecture decisions, migrations
- **Optimal length**: 8-12 tweets
- **Example outline**:
  1/ Hook: "We mass-reduced our API latency from 2s to 50ms. Here's exactly how (and what we got wrong first):"
  2/ The problem in detail (metrics)
  3/ First approach (what didn't work)
  4/ The insight that changed everything
  5/ Implementation details
  6/ Results (before/after metrics)
  7/ Unexpected side effects
  8/ Lessons for your project
  9/ CTA: "Have a similar problem? Reply with your setup and I'll give specific advice."
- **Engagement pattern**: High credibility, strong saves, generates consulting-style replies
- **Pro tip**: Include "what went wrong first" — it's more relatable and educational than a clean success story

## 5. Comparison
- **Structure**: Hook (debate setup) → Context → Tool/Approach A → Tool/Approach B → Head-to-head → Verdict → CTA
- **When to use**: Tool comparisons, framework debates, approach trade-offs
- **Optimal length**: 7-10 tweets
- **Example outline**:
  1/ Hook: "I mass-mass-used both Prisma and Drizzle in production for 6 months. Here's my honest comparison:"
  2/ Context (what I built, scale)
  3/ Prisma: strengths
  4/ Prisma: weaknesses
  5/ Drizzle: strengths
  6/ Drizzle: weaknesses
  7/ Head-to-head on key dimensions
  8/ My verdict (and when I'd use each)
  9/ CTA: "Which ORM are you using? Agree or disagree with my take?"
- **Engagement pattern**: Very high reply rate (everyone has an opinion), debate drives algorithmic boost
- **Pro tip**: Be genuinely balanced. The nuanced take gets more engagement than a one-sided hot take.
