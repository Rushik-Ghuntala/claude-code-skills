# Post Formats

## 1. Value Bomb
- **Description**: Single actionable tip with immediate utility
- **Template**: "[Pattern interrupt]. [Better alternative]. [Why/proof]. [CTA]."
- **Character target**: 200-280
- **When to use**: Code tips, tool recommendations, workflow improvements
- **Example**: "Stop using `console.log`. Use `console.table()` instead. It formats arrays and objects as a table. Try it."
- **Engagement pattern**: High bookmarks (save for later), moderate shares, steady likes
- **Best posting time**: Weekday mornings (8-9:30 AM ET)

## 2. Hot Take
- **Description**: Concise controversial opinion that sparks debate
- **Template**: "Unpopular opinion: [controversial statement about common practice]."
- **Character target**: 80-180
- **When to use**: When you have a genuine contrarian perspective with evidence
- **Example**: "Unpopular opinion: TypeScript slows down prototyping more than it prevents bugs."
- **Engagement pattern**: Very high replies (agreement/disagreement), high retweets, polarizing
- **Best posting time**: Lunch or evening (12-1 PM or 5-6:30 PM ET)

## 3. Storytelling Micro
- **Description**: Mini-story in 280 characters or less
- **Template**: "[Time/event setup]. [Action]. [Surprising result]."
- **Character target**: 150-280
- **When to use**: Personal experiences, career moments, project milestones
- **Example**: "I mass-deleted 40K lines of code today. Tests still pass. Best refactor of my career."
- **Engagement pattern**: High likes, emotional engagement, builds personal brand
- **Best posting time**: Evenings and weekends (9-10:30 PM ET, Saturday mornings)

## 4. Question / Poll
- **Description**: Engagement driver that invites direct participation
- **Template**: "[Question about shared experience]? Mine: [your answer]"
- **Character target**: 80-160
- **When to use**: To drive replies, start conversations, learn from audience
- **Example**: "What's the one tool you'd remove from your stack if forced? Mine: Docker Compose"
- **Engagement pattern**: Highest reply rate of all formats, moderate likes, algorithmic boost from replies
- **Best posting time**: Weekday mornings (maximum active users to seed replies)

## 5. Code Snippet
- **Description**: Visual code example with context and explanation
- **Template**: "TIL: [discovery]. [Code example]. [Why it matters]."
- **Character target**: 200-280 (often paired with code screenshot)
- **When to use**: New APIs, lesser-known features, clever patterns, debugging tips
- **Example**: "TIL: JavaScript has `structuredClone()` now. No more `JSON.parse(JSON.stringify(obj))` hacks."
- **Engagement pattern**: Very high bookmarks, high shares among devs, educational value
- **Best posting time**: Weekday mornings (Tue-Thu for maximum dev audience)

## 6. Announcement / Build-in-Public
- **Description**: Project or milestone update with transparency
- **Template**: "[Milestone]. [Metric]. [What's next / lesson learned]."
- **Character target**: 200-280
- **When to use**: Shipping features, reaching milestones, sharing metrics openly
- **Example**: "Just hit 100 GitHub stars on my CLI tool. Took 3 months. Here's what moved the needle:"
- **Engagement pattern**: Supportive engagement, congratulations replies, builds community
- **Best posting time**: Weekday afternoons (5-6:30 PM ET, "end of workday wins")

## 7. Meme / Cultural
- **Description**: Relatable developer humor with engagement bait
- **Template**: "[Relatable dev situation]. [Humorous twist/punchline]."
- **Character target**: 80-180
- **When to use**: Fridays, weekends, when audience needs levity, universally relatable moments
- **Example**: "Me: 'This will take 2 hours.' The CSS: 'Hold my beer.'"
- **Engagement pattern**: Very high retweet rate, broad appeal, may not convert to followers
- **Best posting time**: Fridays and weekends
